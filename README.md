Stanford simulated a town. **Inception simulates reality.**

## What It Does

Inception drops AI agents into the real world — not a fictional sandbox. They navigate actual streets, enter real cafés, form relationships, keep secrets, and make decisions no one programmed them to make.

Each agent runs a full cognitive architecture:

- **Episodic Memory** — every observation stored, timestamped, scored by recency/importance/relevance
- **Executive Function** — weekly plans generated by LLM, recursively decomposed into sub-actions anchored to real coordinates
- **Allostatic Regulation** — health, vibe, and wealth indices decay each tick; behavior shifts under accumulated load
- **Plausibility Gating** — every planned action is normalized and validated before execution
- **Grounded Reality** — agents walk to Le Consulat, 18 Rue Norvins, Paris — not "The Café"
- **Social Cognition** — relationship decay, discovery graphs, text-native NPC interaction

## Deployment

### Vercel (web)

1. Set **Root Directory** to `web/` in Vercel project settings
2. Add environment variables:
   - `NEXT_PUBLIC_MAPBOX_TOKEN` — your Mapbox public token
   - `NEXT_PUBLIC_SERVER_URL` — your server URL (e.g. `https://api.inception.computer`)

### DigitalOcean App Platform (server)

1. Set **Source Directory** to `/server`
2. Add environment variables:
   - `PORT` — defaults to 8080
   - `MONGO_URI` — MongoDB connection string
   - `OPENROUTER_API_KEY` — OpenRouter API key
   - `FAL_KEY` — Fal.ai API key

### Rename Guardrails

- **DB model/schema names** — do not rename. Field names like `heartbeat_count`, `active_heartbeat_id`, `vibe_index` are persisted in MongoDB.
- **API route paths** — do not rename paths in `server/src/api/runtime/routes.ts` without updating the corresponding client calls in `web/src/lib/api/simulation.ts` and `web/src/lib/api/world.ts` in the same commit.
- **Socket event names** — `heartbeat_update`, `choices_ready`, `runtime_status` are consumed by the frontend layout; rename server + client together.
